# 答辩演示操作手册

## 一、修改现场 IP（换教室/换热点时必做）

只需改一处：

```
web-dashboard/.env.development
```

```
VITE_AI_SERVER_HOST=192.168.x.x   ← 改成当前电脑的局域网 IP
VITE_AI_SERVER_PORT=8000
VITE_CAM_STREAM_PROXY=false
```

改完后重启前端开发服务器（`npm run dev`）即生效，无需改任何代码文件。

---

## 二、答辩当天启动顺序

> **顺序错误** 是现场最常见问题，请严格按以下顺序操作。

### 第 1 步：确认网络

- 路由器 / 热点已启动并正常
- 演示电脑已连接到同一网络
- 确认演示电脑当前 IP（`ipconfig`），对比 `.env.development` 中的配置

### 第 2 步：启动推理服务（server.py）

```bash
cd D:\ultralytics
python server.py
```

等待控制台出现以下字样再进行下一步：

```
INFO:     Application startup complete.
```

> 若出现 `model warmup done` 更好；若只出现 Uvicorn 就绪也可继续。

### 第 3 步：给 ESP32-S3 上电

- 上电后观察 OLED 屏，等待显示 **WiFi OK** 和 **MQTT OK**
- 若 MQTT 持续显示 OFFLINE，等待 15 秒后重试（限流机制）

### 第 4 步：给 ESP32-CAM 上电

- 上电后等待串口监视器（或 OLED）出现 `[NET] WiFi connected`
- ESP32-CAM 启动后会自动开始向 server.py 发送推理请求

### 第 5 步：打开浏览器

```
http://localhost:5173
```

正常状态下，页面应在 10 秒内显示：
- Header 中：**设备在线**（绿色）+ **推理在线**（绿色）
- 左侧画面出现实时流或最新采集帧
- 右侧出现 YOLO 识别结果图

## 三、常见故障速查

| 现象 | 可能原因 | 处理方法 |
|------|----------|----------|
| Header 显示"设备离线" | ESP32-S3 未上电，或 MQTT 未连上 | 检查 S3 OLED 状态；若 WiFi OK 但 MQTT OFFLINE，等待重连 |
| Header 显示"推理离线" | server.py 未启动，或 IP 配置错误 | 检查 `.env.development` 中的 IP 是否正确，重启 `npm run dev` |
| 左侧显示"降级帧"而非实时流 | ESP32-CAM 未上电，或首次推理未完成 | 等待 CAM 上电后 1~2 个推理周期（约 3 秒）后自动切换 |
| 左侧画面卡住 | MJPEG 连接中断 | 页面会自动重连；也可点击"手动刷新"按钮 |
| 舵机无响应 | 置信度阈值过高或识别失败 | 检查页面"识别结果"列；若一直显示"无目标"，尝试在设置里降低置信度 |
| 设置保存失败（推理服务） | server.py 未启动 | 先确保推理服务正常（Header 推理在线为绿色） |
| 设置保存后设备未同步 | ESP32-S3 MQTT 断线 | 阈值已保存到 server.py；S3 重连后会通过 MQTT 收到同步 |

## 四、各组件说明

```
server.py（PC 端）
  ↑ HTTP POST /infer（JPEG 图片）
ESP32-CAM
  ↓ UART 115200（识别结果文本）
ESP32-S3
  ↓ MQTT TCP 1883
OneNET 云平台
  ↑ HTTPS REST
web-dashboard（浏览器）
  ↑ HTTP 代理（Vite dev server → server.py）
```

## 五、快速参数调整

无需重新烧录固件，直接在页面"系统设置"里调整：

- **识别置信度阈值**：建议 `0.70~0.85`，值越高越不容易误触发
- **满溢重量阈值**：演示时可调低至 `200~500g` 方便展示报警

---

## 六、推荐答辩运行参数模板

> 以下参数组合经过综合考虑，覆盖"流畅演示"和"稳定分拣"两种场景，开场前按需选用。

### 模板 A：稳健演示（默认推荐）

| 参数 | 值 | 说明 |
|------|----|------|
| 识别置信度阈值 | `0.75` | 过滤大多数单帧误判 |
| 满溢重量阈值 | `500 g` | 传感器稳定后触发报警 |
| 舵机确认帧数 | `2`（固件内置） | 连续 2 帧同类别才分拣，防止跳变 |
| CAM 推理间隔 | `~1.5 s`（默认） | 正常推理速度 |

适用场景：评委现场放置物体后 3~4 秒内完成分拣，误触发极少。

---

### 模板 B：快速响应（强调反应速度时使用）

| 参数 | 值 | 说明 |
|------|----|------|
| 识别置信度阈值 | `0.60` | 更容易识别，但误触率略高 |
| 满溢重量阈值 | `300 g` | 较少量即触发报警 |
| 舵机确认帧数 | `2`（固件内置） | 保持连续帧保护 |

适用场景：演示环境光线好、物体摆放标准时使用，响应更快但需确保物体放置姿势正确。

---

### 参数调整步骤

1. 进入页面右上角「系统设置」按钮
2. 修改「识别置信度」和「满溢重量阈值」
3. 点击「保存」，观察页面底部的同步进度提示
4. 保存成功后页面自动刷新，设备约 5 秒内收到新阈值

> 注：舵机确认帧数（`SERVO_CONFIRM_FRAMES = 2`）为固件编译时固定参数，如需修改需重新烧录 ESP32-S3。
